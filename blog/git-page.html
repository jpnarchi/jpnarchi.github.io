¿Qué es Git y GitHub?

Git es un sistema de control de versiones distribuido que permite rastrear cambios en el código fuente durante el desarrollo de software. GitHub es una plataforma basada en la nube que aloja repositorios Git y añade muchas de sus propias características.

Flujo de Trabajo en Git

El flujo de trabajo básico de Git consta de los siguientes pasos:

  1. Clonar/Iniciar un repositorio
  2. Crear/Cambiar a una rama (branch)
  3. Modificar archivos en tu directorio de trabajo
  4. Añadir archivos al área de preparación (staging)
  5. Confirmar cambios (commit)
  6. Enviar cambios a un repositorio remoto (push)
  7. Fusionar cambios (merge) de ser necesario
# Flujo de trabajo típico
git clone https://github.com/usuario/repositorio.git
git checkout -b nueva-caracteristica
# Hacer cambios en los archivos
git add .
git commit -m "Añadir nueva característica"
git push origin nueva-caracteristica

Comandos Esenciales de Git

Estos son los comandos más utilizados en Git, ordenados por su función en el flujo de trabajo:

Comando Descripción Ejemplo
git init Inicializa un nuevo repositorio Git git init
git clone Clona un repositorio existente git clone https://github.com/usuario/repo.git
git status Muestra el estado del directorio de trabajo git status
git add Añade archivos al área de preparación git add archivo.txt o git add .
git commit Guarda los cambios en el repositorio git commit -m "Mensaje descriptivo"
git push Envía cambios a un repositorio remoto git push origin main
git pull Obtiene cambios de un repositorio remoto git pull origin main
git branch Lista, crea o elimina ramas git branch nueva-rama
git checkout Cambia entre ramas o restaura archivos git checkout nombre-rama
git merge Fusiona una rama con otra git merge feature

Comandos Adicionales Útiles

  • git log: Muestra el historial de commits
  • git diff: Muestra cambios entre commits
  • git reset: Deshace cambios en el área de preparación
  • git stash: Guarda cambios temporalmente
  • git tag: Gestiona etiquetas para versiones importantes

Flujo de Trabajo con GitHub

GitHub añade capacidades adicionales al flujo de trabajo de Git:

Pasos para colaborar en GitHub:

  1. Fork: Crear una copia del repositorio en tu cuenta
  2. Clone: Clonar el fork a tu máquina local
  3. Branch: Crear una rama para tus cambios
  4. Commit: Realizar cambios y confirmarlos
  5. Push: Enviar cambios a tu fork
  6. Pull Request: Solicitar que tus cambios se incorporen al repositorio original
# Colaboración en GitHub
# 1. Fork (desde la interfaz de GitHub)
# 2. Clonar tu fork
git clone https://github.com/tu-usuario/repositorio.git
# 3. Crear una rama
git checkout -b nueva-caracteristica
# 4. Hacer cambios y commit
git add .
git commit -m "Añadir nueva característica"
# 5. Push
git push origin nueva-caracteristica
# 6. Pull Request (desde la interfaz de GitHub)
